## Top 3 User Needs

Pick only 3 needs that matter most to your target user. For each, write one sentence explaining WHY it matters to this specific user.

1. Different categories for different events (E.g. Funerals, birthday, Christmas)
   - We don't want people to crash out when they see a suggestion that is completely different from your idea.

2. Lists and Notes station
   - Sometimes people need to note things such as who to invite and when to schedule and sometimes keep them hidden. 

3. A simulation planner.
   - Several families would want to picture how to party is going to happen. So why not add a 3D party simulation showing what would happen if they were to have a party at that specific time and day.

## MVP Scope

### Must-Have
*Features required for the core interaction loop: Create event → Add details → Manage guests → Plan activities → Preview/share*

- A "party simulator"
- Note stations
- Content library (Place where you can see different categories)

### Nice-to-Have
*Features that enhance but are not essential*

- Appropriate images and GIF
- Fun design
- AI bot that suggests things

### Cut
*Features you are explicitly NOT building*

- Useless buttons
- Random GIFs
- Scam buttons

## Technical Scaffolding Level

Choose one:

- [ ] **Level A (Starter)** — Simple form + basic list: event creation form, plain guest list, placeholder theme, no drag-and-drop
- [✅] **Level B (Standard MVP)** — Full interaction loop: event creation flow, guest CRUD, activity scheduler, theme selector, responsive layout
- [ ] **Level C (Stretch)** — Everything in B plus: drag-drop activities, theme previews with images, shareable event link, print-friendly view
